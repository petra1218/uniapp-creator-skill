# 前端页面模式

## 用法
先判断页面模式，再判断复用层级。不要先做抽象设计系统。

## 页面模式表

| 页面模式 | 默认方向 | 优先复用 | 设计重点 | 何时允许整体实现 |
|---|---|---|---|---|
| 列表/筛选/表单/详情等业务操作页 | 移动优先，Web 增强 | 官方组件、项目内已有官方封装 | 信息密度、操作效率、状态清晰 | 官方组合无法承载关键交互时 |
| 工作台/首页聚合页 | 移动优先，Web 做分栏增强 | 官方卡片、栅格、导航组件 | 快速进入高频任务 | 需要明显品牌化但仍保留官方基础件时 |
| 内容展示/活动/营销页 | 移动竖屏与 Web 横屏同时考虑 | 官方基础组件 + 统一风格包装 | 首屏表达、图文节奏、跨端阅读体验 | 页面表现高度依赖整体视觉时 |
| 管理后台页 | Web 优先，同时考虑移动最小可用 | `uni-admin` 或项目内后台封装 | 表格、筛选、权限、批量操作 | 官方后台结构无法覆盖核心业务流时 |

## 复用层级

按顺序判断：

1. 直接复用官方模块
2. 官方模块最小改造
3. 官方基础件 + 薄包装
4. 整体实现

## 风格统一

默认要求：
- 用 `ui-ux-pro-max` 统一风格
- 能用官方 Sketch 资源确定页面骨架时，先沿用信息结构，再做最小视觉改造
- 有原型图时，先把原型风格与现有页面统一
- 没有原型图时，不额外发明一套复杂设计语言
