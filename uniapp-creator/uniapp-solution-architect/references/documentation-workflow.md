# 文档工作流

## 读文档顺序

1. `docs/project-system.md`
2. 当前任务卡
3. 相关设计文档
4. 如涉及数据，再看数据文档
5. 如涉及流程或平台分支，再看流程文档

## 设计输出至少要落下什么

- 总体设计
- 技术约定
- 开发清单
- 当前轮次下一步
- 数据文档要求（如有）
- Mermaid 要求（如有）

## 执行完成后至少回写什么

1. 任务卡执行记录
2. `docs/project-system.md`
3. `docs/tracking/change-log.md`
4. 如有数据变化，更新数据文档
5. 如有流程变化，更新流程文档

## 如何和后续执行衔接

设计文档里必须直接写明：
- 下一步做哪个模块或步骤
- 适合先 `writeplan` 还是直接 `execplan`
- 哪些步骤更适合 TDD
