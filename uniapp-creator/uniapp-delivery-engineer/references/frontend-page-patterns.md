# 前端页面模式

## 执行时怎么用

先判断当前页面属于哪类，再决定是复用官方模块、最小改造，还是整体实现。

## 页面模式表

| 页面模式 | 默认方向 | 优先策略 |
|---|---|---|
| 业务操作页 | 移动优先，Web 增强 | 官方模块改造优先 |
| 工作台/聚合页 | 移动优先，Web 分栏增强 | 官方基础件组合优先 |
| 内容展示/营销页 | 同时考虑移动竖屏与 Web 横屏 | 统一视觉，必要时整体实现 |
| 后台管理页 | Web 优先 | 优先沿用 `uni-admin` 或现有后台结构 |

## 复用层级

按顺序判断：

1. 直接复用官方模块
2. 官方模块最小改造
3. 官方基础件 + 薄包装
4. 整体实现

## 风格提醒

- 默认用 `ui-ux-pro-max` 做风格统一
- 如存在官方 Sketch 资源或已确认原型，优先对齐页面骨架，再做样式落地
- 不为了设计感推翻现有可复用结构
- 先保证好用，再做视觉增强
